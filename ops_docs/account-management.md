## 一、用户自助账号服务

### 1.1 账号冻结怎么处理

**现象**：登录系统时提示「账号已冻结」「Account Frozen」或「用户状态异常」。

**解决方案**（课题标准流程）：

1. **自助解冻（推荐）**：访问自助服务网址 http://XXX.YYY.ZZZ ，使用手机号或企业账号登录，在「账号服务 → 账号解冻」中提交申请，通常 10 分钟内生效。
2. **电话热线**：拨打运维服务热线 **400-8800-1234**（工作日 8:30–18:00），提供工号与姓名，人工核实后解冻。
3. **工单申告**：登录运维申告门户，提交「账号解冻」工单，运维人员在后台处理。

**常见冻结原因**：

- 连续输错密码超过 5 次（部分系统会自动冻结而非仅锁定）
- 长期未登录（超过 90 天）
- 安全策略触发（异常登录地点、批量下载敏感数据）
- 管理员主动冻结（离职交接、违规调查期间）

**注意**：账号「冻结」与「锁定」不同——锁定多为临时（约 30 分钟），冻结需主动解冻。

---

### 1.2 账号解冻流程

**自助解冻步骤**：

```
1. 浏览器访问 http://XXX.YYY.ZZZ
2. 点击「账号解冻」→ 输入工号 / 手机号
3. 完成短信验证码校验
4. 阅读并确认《账号安全承诺书》
5. 提交申请 → 等待短信通知「解冻成功」
```

**后台人工解冻步骤**（运维人员）：

```
1. 登录运维申告门户后台
2. 进入「账号管理 → 用户账号查询」
3. 搜索目标用户，确认冻结原因
4. 核实报障人身份（工号 + 部门 + 手机验证码）
5. 执行解冻操作并填写处理备注
6. 电话回访报障人确认可正常登录
```

---

### 1.3 如何重置密码

**自助重置**：

1. 访问 http://XXX.YYY.ZZZ/password-reset
2. 选择「忘记密码」
3. 通过企业微信扫码或短信验证码验证身份
4. 设置新密码（至少 8 位，含大小写字母、数字、特殊符号中至少三类）

**无法自助时**：

- 拨打热线 **400-8800-1234**
- 或在运维申告门户提交「密码重置」工单

**密码策略**：

| 规则 | 要求 |
|------|------|
| 最小长度 | 8 位 |
| 复杂度 | 大小写 + 数字 + 特殊符号至少三类 |
| 历史密码 | 不能与最近 5 次相同 |
| 有效期 | 90 天强制更换 |

---

### 1.4 如何创建用户账号（新员工开通）

**标准流程**：

1. HR 系统在入职审批通过后，自动触发账号开通流程
2. 系统在入职日上午 9:00 前创建域账号（AD 账号）
3. 初始密码通过短信发送至员工手机
4. 员工首次登录须修改密码并绑定 MFA

**加急开通**（入职当天上午仍未收到账号）：

1. 部门助理在 HR 系统确认入职流程已完成
2. 提交 ITSM 工单「新员工账号加急开通」，附工号与入职日期
3. IT 运维在 2 小时内处理并短信通知

**账号格式**：

- 域账号：`姓名拼音` 或 `姓名拼音 + 数字`（重名时）
- 邮箱：`姓名拼音@company.com`

## 二、运维人员账号管理（后台门户）

> 本节对应运维申告门户后台的账号 CRUD 功能，供一线运维人员操作参考。

### 2.1 如何创建运维账号

**前置条件**：申请人已填写《运维人员权限申请表》，并经部门负责人审批。

**操作步骤**：

```
1. 登录运维申告门户后台（http://portal.company.com/admin）
2. 进入「系统管理 → 运维账号管理」
3. 点击「新增账号」
4. 填写必填信息：
   - 用户名（唯一，建议用工号）
   - 密码（系统生成或手动设置，须符合密码策略）
   - 真实姓名
   - 联系电话
5. 点击「保存」→ 系统返回「账号创建成功」
```

**API 参考**（供联调）：

```http
POST /api/operators
Content-Type: application/json

{
  "username": "ops_wangxm",
  "password": "TempPass@2026",
  "realName": "王小明",
  "phone": "13800138000"
}
```

**注意**：密码在数据库中以 BCrypt 哈希存储，明文不可查看。

---

### 2.2 如何冻结运维账号

**适用场景**：运维人员离职、长期休假、权限违规、账号疑似泄露。

**操作步骤**：

```
1. 登录后台 →「运维账号管理」
2. 搜索目标账号
3. 点击「冻结」→ 确认操作
4. 账号 isActive 设为 false，该账号无法登录后台
5. 在工单系统中记录冻结原因与操作人
```

**API 参考**：

```http
DELETE /api/operators/{id}
```

**说明**：此为软删除（冻结），数据保留，可后续恢复。

---

### 2.3 如何修改运维账号信息

**可修改字段**：真实姓名、联系电话、密码。

**操作步骤**：

```
1. 后台搜索目标账号
2. 点击「编辑」
3. 修改所需字段（修改密码须重新加密存储）
4. 保存 → 返回「账号修改成功」
```

**API 参考**：

```http
PUT /api/operators/{id}
Content-Type: application/json

{
  "realName": "王小明",
  "phone": "13900139000",
  "password": "NewPass@2026"
}
```

---

### 2.4 如何查询运维账号

**查询方式**：

1. **列表查询**：后台「运维账号管理」默认展示所有未冻结账号（`isActive=true`）
2. **按用户名搜索**：输入用户名精确查找
3. **API 查询**：

```http
GET /api/operators
```

**返回字段**：id、username、realName、phone、isActive（密码字段不返回）

## 三、账号安全与异常处理

### 3.1 账号异常登录告警怎么处理

**现象**：收到短信「您的账号在异常地点登录」或安全平台告警。

**处理步骤**：

1. 若非本人操作，立即拨打 **400-8800-1234** 冻结账号
2. 修改密码并重新绑定 MFA
3. 检查近期操作日志：`last -20`（Linux）或 AD 审计日志
4. 提交安全事件工单，由安全团队溯源

---

### 3.2 MFA 多因素认证绑定与解绑

**绑定步骤**：

1. 下载 Microsoft Authenticator 或 Google Authenticator
2. 登录 http://XXX.YYY.ZZZ/security →「绑定 MFA」
3. 扫描二维码，输入 6 位动态验证码确认

**解绑步骤**（需人工）：

1. 提交 ITSM 工单或拨打热线
2. 提供工号、身份证明
3. 管理员重置 MFA 绑定，用户重新绑定新设备

---

### 3.3 共享账号与临时账号管理

**规定**：禁止使用共享账号进行日常运维操作。

**临时账号**（访客/外包）：

- 最长有效期 7 天
- 由外包管理员或部门负责人申请
- 到期自动禁用，数据保留 30 天后删除
- 续期须重新审批

## 四、常见问题速查

### 4.1 登录提示 Error 401 / 403

| 错误码 | 含义 | 处理 |
|--------|------|------|
| 401 Unauthorized | 未认证或 Token 过期 | 重新登录，检查 Token 有效期 |
| 403 Forbidden | 已认证但无权限 | 确认账号权限，联系管理员开通 |

---

### 4.2 域账号与本地账号的区别

| 类型 | 用途 | 管理方式 |
|------|------|----------|
| 域账号（AD） | 登录电脑、WiFi、邮箱、内网系统 | IT 统一管控，SSO 单点登录 |
| 本地账号 | 仅限单台电脑本地登录 | 不推荐日常使用 |

**建议**：日常办公一律使用域账号。
